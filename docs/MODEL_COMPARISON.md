## 内存占用详解

### 内存计算公式
```
总内存 = 模型文件大小 × 1.2~1.5 + 上下文内存 + 系统开销
```

### 各模型实际内存占用

| 模型 | 模型文件 | 运行时内存 | 峰值内存 | 建议总内存 |
|------|----------|------------|----------|------------|
| Qwen2.5 1.5B | 1GB | 1.2GB | 1.5GB | 4GB |
| Gemma 3 4B | 2.6GB | 3.2GB | 4GB | 6GB |
| Qwen2.5 3B | 1.9GB | 2.4GB | 3GB | 6GB |
| Llama 3.2 3B | 1.9GB | 2.4GB | 3GB | 6GB |
| Phi-4 14B | 8GB | 10GB | 12GB | 16GB+ |

### 内存优化技巧

1. **减小上下文长度**
   ```python
   config = LLMConfig(context_length=2048)  # 默认8192
   ```

2. **使用更激进的量化**
   - Q4_K_S 比 Q4_K_M 节省 10-15% 内存
   - Q3_K_M 比 Q4_K_M 节省 20-25% 内存（质量略有下降）

3. **减少批处理大小**
   ```python
   config = LLMConfig(batch_size=256)  # 默认512
   ```

## 选择建议

### 场景 1：纯中文法律文档处理
**推荐：Qwen2.5 3B 或 Gemma 3 4B**
- 中文优化最好
- 法律术语理解准确
- 案号、金额提取准确

### 场景 2：资源受限（4-6GB内存）
**推荐：Qwen2.5 1.5B**
- 内存占用最小
- 速度最快
- 中文效果依然不错

### 场景 3：中英混合文档
**推荐：Gemma 3 4B 或 Llama 3.2 3B**
- 多语言支持好
- 格式处理规范

### 场景 4：复杂推理分析
**推荐：Phi-4 14B（需要高配）或 Gemma 3 4B**
- 推理能力强
- 可以处理复杂逻辑

### 场景 5：长文档分析
**推荐：Gemma 3 4B**
- 128K 上下文
- 可以处理整份合同
